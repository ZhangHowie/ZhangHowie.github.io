前言

查看源码遇到的问题总结

查看源码总是能遇到这样或者那样的问题，我就拿出自己遇到的几个问题，记录下来，以后有什么问题再增加解决方案。

首先是应对All packages are not available for download

首先点开Handler源码会提醒很多Throws

这是因为你的compileSdkVersion中没有包含当前版本所需要的Sources for Android，

例如目前还没有最新版Oreo的Sources，你却把当前的compileSdkVersion设成了26，那么你就会报这样的错误。

那么你就可以将compileSdkVersion调整为你已经下载Sources的SDK对应的数字，就可以解决了。

另外，“RuntimeException(“Stub!”)” 表示实际运行时的逻辑会由 Android ROM 里面相同的类代替执行。

应对decompiled .class file, bytecode version:52.0(Java 8)

前几天更新了Java版本到java8，然后查看源码的时候，就提示，

decompiled .class file, bytecode version:52.0(Java 8)
网上查了一下资料，是Android SDK没有刷新造成的，
现在需要点开Preferences –> Appearance & Behavior –> Android SDK –>edit

就编辑一下，然后下一步，会让你更新一个包，更新完成后，问题就解决了，就可以继续开心的查看源码了。
